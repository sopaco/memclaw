# System Boundary Interface Documentation

This document describes the system's external invocation interfaces, including CLI commands, API endpoints, configuration parameters, and other boundary mechanisms.

## Command Line Interface (CLI)

### cortex-mem-cli vector prune

**Description**: Vector index pruning command for maintenance. Removes outdated or low-quality vectors from the index.

**Source File**: `context-engine/index.ts (runMaintenance function)`

**Arguments**:

- `configPath` (string): required - Configuration file path for cortex-mem-cli
- `tenantId` (string): optional - Tenant identifier for memory isolation (default: `tenant_claw`)
- `timeout` (number): optional - Command execution timeout in milliseconds (default: `300000`)

**Usage Examples**:

```bash
executeCliCommand(['vector', 'prune'], configPath, tenantId, 300000)
```

### cortex-mem-cli vector reindex

**Description**: Vector index reindexing command for maintenance. Rebuilds vector indexes for improved search performance.

**Source File**: `context-engine/index.ts (runMaintenance function)`

**Arguments**:

- `configPath` (string): required - Configuration file path for cortex-mem-cli
- `tenantId` (string): optional - Tenant identifier for memory isolation (default: `tenant_claw`)
- `timeout` (number): optional - Command execution timeout in milliseconds (default: `300000`)

executeCliCommand(['vector', 'reindex'], configPath, tenantId, 300000)

### cortex-mem-cli layers ensure-all

**Description**: Memory layer management command. Ensures all L0/L1/L2 memory layers are properly created and maintained.

**Source File**: `context-engine/index.ts (runMaintenance function)`

**Arguments**:

### openclaw plugins install memclaw

**Description**: OpenClaw CLI command to install the MemClaw plugin into the OpenClaw ecosystem.

**Source File**: `plugin/index.ts (installation documentation)`

**Usage Examples**:

```bash

```

## API Interfaces

### HTTP REST (GET/POST/PUT/DELETE) http://localhost:8085 (configurable via serviceUrl)

**Description**: CortexMemClient HTTP client interface for communicating with cortex-mem-service REST API. Handles L0/L1/L2 tiered memory retrieval, search, browsing, and session management.

**Source File**: `plugin/src/client.ts, context-engine/client.ts`

**Request Format**: JSON

**Response Format**: JSON

**Authentication**: API Key (configured in config.toml)

### Function calls via PluginAPI interface OpenClaw Plugin System (internal)

**Description**: OpenClaw Plugin API interface for plugin integration. Provides registerTool, registerService, registerHook, registerContextEngine methods for plugin lifecycle management.

**Source File**: `plugin/index.ts, context-engine/index.ts`

**Request Format**: TypeScript interfaces (PluginLogger, ToolDefinition, PluginAPI)

**Response Format**: TypeScript return values

**Authentication**: Plugin API registration

## Integration Suggestions

### Plugin Configuration

OpenClaw Plugin Integration - Register MemClaw as a plugin in openclaw.json configuration file with proper slot assignment for memory and context engine.

**Example Code**:

```
{
  "plugins": {
    "entries": {
      "memclaw": {
        "enabled": true,
        "config": {
          "serviceUrl": "http://localhost:8085",
          "tenantId": "tenant_claw",
          "autoStartServices": true
        }
      }
    },
    "slots": {
      "memory": "none",
      "contextEngine": "memclaw-context-engine"
    }
  }
}
```

**Best Practices**:

- Always validate config.toml exists before starting services
- Ensure Qdrant and cortex-mem-service binaries are available before auto-start
- Use tenant isolation for multi-user deployments
- Configure appropriate timeout values for CLI commands (default 300000ms)
- Enable autoStartServices only in development environments

### Configuration Setup

Configuration Management - Set up config.toml with required LLM and embedding API keys. The system auto-generates this file on first run and opens it for editing.

**Example Code**:

```
# config.toml
[llm]
api_base_url = "https://api.openai.com/v1"
api_key = "your-api-key-here"
model_efficient = "gpt-5-mini"

[embedding]
api_base_url = "https://api.openai.com/v1"
api_key = "your-api-key-here"
model_name = "text-embedding-3-small"
```

**Best Practices**:

- Store API keys securely, never commit to version control
- Use environment variables for sensitive configuration in production
- Validate llm.api_key and embedding.api_key before service startup
- Consider using separate API keys for LLM and embedding services
- Review config.toml template for all required fields on first run

### Service Management

Service Orchestration - Manage Qdrant and cortex-mem-service lifecycle. Services can be auto-started or manually controlled via registerService API.

**Example Code**:

```
api.registerService({
  id: 'memclaw-context-engine',
  start: async () => {
    await ensureAllServices(log);
    await client.switchTenant(config.tenantId);
  },
  stop: async () => {
    stopAllServices(log);
  }
});
```

**Best Practices**:

- Run maintenance tasks during low-usage periods
- Monitor service health with checkServiceStatus before operations
- Implement proper error handling for CLI command failures
- Use the 3-hour maintenance interval as a baseline, adjust based on usage
- Log all maintenance operations for troubleshooting

### Tool Integration

Tool Registration - Register memory tools with OpenClaw via registerTool API. Tools provide semantic search, memory retrieval, and session management capabilities.

**Example Code**:

```
const tools = createTools(client, toolConfig, api.logger);
for (const [name, tool] of tools) {
  api.registerTool(tool, { name });
}
```

**Best Practices**:

- Register tools with descriptive names for better agent understanding
- Implement proper error handling in tool execute functions
- Use optional flag for non-critical tools
- Document tool parameters clearly for agent consumption
- Test tools independently before plugin integration
